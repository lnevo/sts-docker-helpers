# Switch list building — checkpoint (2026-07-08)

Phased master switch lists for **D749**, **NVL**, and **CK1** are generated by dry-running session operations inside a DB transaction, then rendering HTML (mobile + half sheet) under `switchlists/session_N/`.

## Pipeline (correct order)

1. **Warm start** (`apply_warm_start.sh --tracked`) — simulates end-of-prior-session state: locals, CK1 weigh, STG-DEMMLER bookend, STG-SCULLY backlog at Scully.
2. **Begin session** (`begin_session.sh --run-stg-scully`) — STG-SCULLY, increment session, **fill unfilled orders**, reposition empties, **auto-assign jobs**.
3. **Generate switch lists** (`generate_switchlists.sh --format=phased`) — dry-run job phases, write HTML + root index.

`run_session_simulations.sh` runs steps 1→2→3 each iteration. Switch lists are for the **operating session after begin_session** (session N+1 when warm start ends at N).

`begin_session.sh --switchlists` runs step 2 then 3 for the live DB.

## Job phase logic (summary)

### NVL (3 phases) — best coverage

1. **Scully → South Yard** — `warm_start_begin_operating_session()` (STG-SCULLY, fill, assign) + NVL Scully pickup.
2. **Neville Island Industries** — Demmler setout at South, island/Shenango pickups only.
3. **South Yard → Scully** — Island work, D749/CK1 Scully handoffs staged at South, all Scully-bound on train.

### D749 (3 phases) — partial at session open

1. **Demmler → South Yard** — cars currently assigned to D749.
2. **South Yard → Demmler** — Demmler handoffs spotted at South (may need prior-session staging).
3. **Demmler → South Yard** — return loads after simulated Demmler round trip.

Legs 2–3 are empty when the DB has no Demmler handoffs at South at session-open snapshot. Full 3-leg lists appear after warm-start sessions 2+ (pre-begin_session end state had richer handoffs than post-begin_session session 5).

### CK1 (3 phases)

1. **South Yard → Shenango** — reload/outbound coke at South.
2. **Shenango → South Yard Scale** — loaded coke at plant.
3. **South Yard setouts** — after weigh simulation in dry-run.

Phase 1 is often empty when coke is already at Shenango from the prior sim cycle.

## Commands

```bash

./bin/run_session_simulations.sh --sessions 4

# From sim_session_4 backup → operate session 5
./bin/apply_hart_seed.sh --sql-file ~/sts/sts-backups/sim_session_4
./bin/begin_session.sh --run-stg-scully --switchlists

# Lists only (current DB)
./bin/generate_switchlists.sh --format=phased --session=N
```
